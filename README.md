# Obsidian-Claude Automation Agent

A Python tool that automatically processes natural language requests embedded in Obsidian notes using Claude AI via the Obsidian MCP (Model Context Protocol) server.

## Overview

This agent scans your Obsidian vault for notes containing `@claude` requests, sends them to Claude AI with restricted tool permissions, and creates linked response notes—all running as a scheduled background task.

**Status:** 🚧 Under Development - MCP integration complete, core features in progress

## Features

- 🔍 **Smart Note Scanning** - Finds notes modified in the last week containing `@claude` requests
- 🤖 **Claude AI Integration** - Processes requests using Claude API with configurable tool restrictions
- 📝 **Response Notes** - Creates linked markdown notes with Claude's responses
- 🔒 **Security** - Restricts Claude's access to approved tools only (no bash execution, no note modifications)
- ⏱️ **Rate Limiting** - Enforces 5 requests/hour limit to control API usage
- 🔄 **Deduplication** - Prevents re-processing of completed requests
- 📅 **Scheduled Execution** - Designed to run via cron/systemd/Task Scheduler

## Quick Start

### Prerequisites

- Python 3.10+
- Node.js/npm (for Obsidian MCP server)
- Claude API key
- Obsidian vault

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/aseeda/obsidian_claude.git
   cd obsidian_claude
   ```

2. **Install Python dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Install Obsidian MCP server**
   ```bash
   npx @mauricio.wolff/mcp-obsidian@latest --help
   ```

4. **Configure the agent**
   - Edit `config/default_config.yaml` with your vault path and settings
   - Set `ANTHROPIC_API_KEY` environment variable

### Configuration

**Main Config (`config/default_config.yaml`):**
```yaml
mcp:
  server_command: "npx"
  server_args:
    - "@mauricio.wolff/mcp-obsidian@latest"
    - "/path/to/your/vault"  # Update this!
  timeout: 30
  max_retries: 3

claude:
  api_key_env: "ANTHROPIC_API_KEY"
  model: "claude-sonnet-4"
  max_tokens: 4096
  temperature: 0.7

scanning:
  modified_within_days: 7
  check_interval_seconds: 300

rate_limit:
  max_requests_per_hour: 5
```

**Permissions (`config/vault_permissions.yaml`):**
```yaml
default:
  allowed_tools:
    - read_note
    - search_notes
    - write_note
    - web_search
    - web_fetch
```

## Usage

### Request Syntax

Add `@claude` requests to your Obsidian notes:

```markdown
@claude Summarize the key points from this meeting note

@claude: What are the action items I need to complete?

@claude - Create a list of questions based on this research

@claude """
Analyze this project plan and suggest:
1. Potential risks
2. Missing tasks
3. Timeline improvements
"""
```

### Response Format

The agent creates response notes like `your_note_response_20260305_143045.md`:

```markdown
# Claude Response

**Source Note:** [[your_note]]
**Request:** Summarize the key points from this meeting note
**Timestamp:** 2026-03-05 14:30:45
**Status:** Success

---

## Response

[Claude's response appears here]

---

*Generated by Obsidian-Claude Agent*
```

After processing, `@claude` is replaced with `@claude-done` in the source note.

### MCP Client Usage

The MCP client uses async/await patterns:

```python
import asyncio
from src.mcp_client import MCPClient

async def main():
    client = MCPClient(
        server_command="npx",
        server_args=["@mauricio.wolff/mcp-obsidian@latest", "/path/to/vault"]
    )

    async with client:
        # Search for notes
        notes = await client.search_notes(query="@claude")

        # Read a note
        content = await client.read_note("path/to/note.md")

        # Create response note
        await client.create_note(
            path="response.md",
            content="# Response\n\nContent here"
        )

asyncio.run(main())
```

## Security & Privacy

- **No Code Execution:** Claude cannot run shell commands via `bash` tool
- **Read-Only by Default:** Claude cannot modify existing notes (only create new ones)
- **Tool Allowlist:** Only approved MCP tools are accessible
- **Local Processing:** All data stays on your machine except API calls to Anthropic
- **API Key Security:** Uses environment variables, never hardcoded
