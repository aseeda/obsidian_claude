# Obsidian-Claude Automation Agent

A Python tool that automatically processes natural language requests embedded in Obsidian notes using Claude AI via direct file system access.

## Overview

This agent scans your Obsidian vault for notes containing `@claude` requests, sends them to Claude AI with restricted tool permissions, and creates linked response notes—all running as a scheduled background task.

**Status:** ✅ v1.0 Complete - Fully functional with direct file system access

## Features

- 🔍 **Smart Note Scanning** - Finds notes modified in the last week containing `@claude` requests
- 🤖 **Claude AI Integration** - Processes requests using Claude API with configurable tool restrictions
- 📝 **Response Notes** - Creates linked markdown notes with Claude's responses
- 🔒 **Security** - Restricts Claude's access to approved tools only (no bash execution, no note modifications)
- ⏱️ **Rate Limiting** - Enforces 5 requests/hour limit to control API usage
- 🔄 **Deduplication** - Prevents re-processing of completed requests
- 📅 **Scheduled Execution** - Designed to run via cron/systemd/Task Scheduler

## Quick Start

### Prerequisites

- Python 3.10+
- Claude API key
- Obsidian vault directory

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/aseeda/obsidian_claude.git
   cd obsidian_claude
   ```

2. **Install Python dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure the agent**
   - Copy `.env.example` to `.env` and add your `ANTHROPIC_API_KEY`
   - Edit `config/default_config.yaml` with your vault path

### Configuration

**Main Config (`config/default_config.yaml`):**
```yaml
obsidian:
  vault_path: "/path/to/your/vault"  # Update this!
  cli_path: null  # Auto-detects Obsidian CLI, or uses direct file access
  timeout: 30

claude:
  api_key_env: "ANTHROPIC_API_KEY"
  model: "claude-sonnet-4-5-20250929"
  max_tokens: 4000
  temperature: 0.7

scanning:
  recent_timeframe: 7  # days
  check_interval: 300  # seconds

rate_limit:
  max_requests_per_hour: 5

response:
  max_length: 5000  # characters
  include_timestamp: true
  note_suffix: "_response_"
```

**Permissions (`config/vault_permissions.yaml`):**
```yaml
default:
  allowed_tools:
    - obsidian_read_note
    - obsidian_search_notes
    - obsidian_write_note
```

## Usage

### Request Syntax

Add `@claude` requests to your Obsidian notes:

```markdown
@claude Summarize the key points from this meeting note

@claude: What are the action items I need to complete?

@claude - Create a list of questions based on this research

@claude """
Analyze this project plan and suggest:
1. Potential risks
2. Missing tasks
3. Timeline improvements
"""
```

### Response Format

The agent creates response notes like `your_note_response_20260305_143045.md`:

```markdown
# Claude Response

**Source Note:** [[your_note]]
**Request:** Summarize the key points from this meeting note
**Timestamp:** 2026-03-05 14:30:45
**Status:** Success

---

## Response

[Claude's response appears here]

---

*Generated by Obsidian-Claude Agent*
```

After processing, `@claude` is replaced with `@claude-done` in the source note.

## Security & Privacy

- **No Code Execution:** Claude cannot run shell commands via `bash` tool
- **Read-Only by Default:** Claude cannot modify existing notes with patches (only create new ones)
- **Tool Allowlist:** Only approved tools are accessible (read, search, write)
- **Local Processing:** All data stays on your machine except API calls to Anthropic
- **API Key Security:** Uses environment variables via `.env` file, never hardcoded
- **Direct File Access:** No external dependencies or network servers required
